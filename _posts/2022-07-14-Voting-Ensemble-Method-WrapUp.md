### Introduction


This post contains a basic **Ensemble** method. **Ensemble** method is similar to forming a committee. Several people gather together to produce the best outcome. **Ensemble** method can be 3 different subtypes: **Voting**, **Bagging**, and **Boosting**. This post will talk about how to build the **Voting** Ensemble method using Breast Cancer data from [Kaggle](https://www.kaggle.com/datasets/sarahvch/breast-cancer-wisconsin-prognostic-data-set). 


##### Voting Method


The voting method integrates different types of machine learning models to produce the best outcome. After obtaining different predictions from each machine learning model, there are two ways to get the result, **Hard Voting** and **Soft Voting**. **Hard Voting** is following the majority while **soft voting** is taking the average of the result. **Soft Voting** is widely used.

### Building Ensemble Method


To build the **Ensemble** model, I will use *logistic regression*, *KNeighborsClassifier*, and *Support Vector Machine* as individual models or classifiers. Each model's description will be discussed in a different post.


##### Loading Packages



```python
from sklearn.ensemble import VotingClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.neighbors import KNeighborsClassifier
from sklearn.svm import SVC
```

##### Parameters of Ensemble Method


First, you have to build each model separately. Then, make a list of tuples of models, and use *estimators* parameter to call the list. Lastly, choose type of voting method at the end.



VotingClassifier(estimator = **list of tuple**, voting = **'soft' or 'hard'**)




### Conclusion


It is not always true that an ensemble method will bring the best result. However, the ensemble method generally produces higher results. It allows seeing different perspectives using different estimators inside the model, which could usually bring a better result compared to a single estimator.
